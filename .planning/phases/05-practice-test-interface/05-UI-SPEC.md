## Color

All colors from `app/globals.css` `:root` block — never hardcode hex in components.

| Role | CSS Variable | Hex | Usage |
|------|-------------|-----|-------|
| Dominant (60%) | `var(--bg-primary)` | `#0a0a0a` | Page background, session background |
| Secondary (30%) | `var(--bg-secondary)` / `var(--bg-card)` | `#111111` / `#161616` | Session header bar, question card, results cards, setup card |
| Accent (10%) | `var(--accent)` | `#6366f1` | Reserved for specific elements listed below |
| Destructive | `var(--accent-danger)` | `#ef4444` | Timer warning state, wrong-answer indicators in results |

Accent (`var(--accent)` / `#6366f1`) is reserved for:
- "Start Test" primary CTA button (setup view)
- "Submit Test" primary CTA button (session view)
- "Retake Test" primary CTA button (results view)
- Active/answered question number cell in nav grid (filled background)
- Progress bar fill in session header
- Projected AP score ring fill (conic-gradient, same `.score-ring` pattern)
- Timed mode toggle — ON state indicator
- Timer show/hide toggle — active state

Semantic colors:
- `var(--accent-success)` `#22c55e` — correct answer count badge in session header; correct-answer highlight in missed questions list
- `var(--accent-warning)` `#f59e0b` — flagged-for-review question cells in nav grid
- `var(--accent-danger)` `#ef4444` — wrong answer count badge in session header; timer <5 min warning; wrong-answer highlight in missed questions list; "incorrect" feedback in results
- `color-mix(in srgb, {color} 15%, transparent)` — background tint for all semantic badges (established pattern from MCQSession/MCQResults)
- `color-mix(in srgb, {color} 30%, transparent)` — border for all semantic badges

Question state colors in nav grid:
- Unanswered: `var(--bg-card)` background, `var(--bg-border)` border, `var(--text-muted)` numeral
- Answered: `var(--accent)` background at 20% opacity via `color-mix`, `var(--accent)` border, `var(--text-primary)` numeral
- Flagged for review: `var(--accent-warning)` background at 15% opacity, `var(--accent-warning)` border, numeral in warning color
- Current (active): `var(--accent)` solid background (full opacity), white numeral

## View Specifications

### View 1: Setup (`type TestView = 'setup'`)

Layout: Centered card, max-width 480px, vertically centered in page.

Elements (top to bottom):
1. Subject name as page title — 24px weight 700
2. "Practice Test" subtitle — 14px weight 400, `var(--text-secondary)`
3. Test metadata row — question count + time limit — 14px `var(--text-muted)`, icon + label pairs using Lucide `Clock` and `BookOpen`
4. Divider: 1px `var(--bg-border)`
5. Timed mode toggle row — label "Timed Mode" + toggle switch (checkbox styled as pill toggle); default ON
6. Timer display toggle row — label "Show Timer" + toggle; default ON; disabled (grayed out) when Timed Mode is OFF
7. Divider
8. "Start Test" button — full-width, primary accent style (padding 14px 20px)

Card: `var(--bg-card)` background, `var(--bg-border)` border, `var(--radius-lg)` border-radius, 32px padding.

Toggle pill style: 44px wide × 24px tall; thumb is 20px circle. ON state: track is `var(--accent)`, thumb is white. OFF state: track is `var(--bg-border)`, thumb is `var(--text-muted)`. Transition: 150ms ease.

### View 2: Session (`type TestView = 'session'`)

Layout: Full-height column. Header is sticky at top of content area (not fixed — flows with page scroll after nav).

Session header (sticky, `var(--bg-secondary)` background, 1px bottom border `var(--bg-border)`, padding 12px 16px):
- Left: Subject name + "Practice Test" label — 14px `var(--text-secondary)`
- Center: Timer display (when timed mode ON and show timer ON) — 20px weight 700 monospace, normal color until <5 min then `var(--accent-danger)`; hidden when show-timer toggled off
- Right: Timer show/hide toggle button (Lucide `Eye`/`EyeOff`, 16px icon, `var(--text-muted)`) + score tally badges (correct/wrong — same pattern as MCQSession)

Question navigation panel (below header, above question card on desktop; collapsible drawer on mobile):
- Grid of numbered cells, 10 per row on desktop, 8 per row on mobile
- Each cell: 36px × 44px, `var(--radius-sm)` border-radius, state colors as specified in Color section
- Flag icon (Lucide `Flag`, 12px) overlaid on flagged cells in bottom-right corner, `var(--accent-warning)`
- "Flag for Review" button near question: Lucide `Flag` icon + text, ghost style (no background, `var(--text-muted)` default, `var(--accent-warning)` on active)

Question area (scrollable, max-width 720px, centered):
- Question number indicator: "Question N of M" — 14px `var(--text-muted)`
- MCQCard renders here (reused as-is)
- Navigation row below card: Lucide `ChevronLeft` / `ChevronRight` arrows + "Submit Test" button
  - Prev/Next: ghost style buttons, 44px touch target, `var(--text-secondary)` icon, transitions to `var(--text-primary)` on hover
  - "Submit Test": only visible when all questions answered OR on last question (always accessible via keyboard); primary accent pill style

Mobile layout (< 768px):
- Question nav grid collapses to a horizontal scrolling strip showing current ± 4 question numbers
- Timer moves to below nav strip
- Session header shows only subject label and score badges (timer strip below)

Submit confirmation modal:
- Not a dialog/modal component — inline confirmation within session view
- When "Submit Test" is clicked and questions remain unanswered: inline banner below nav grid
- Banner: `var(--bg-card)` background, `var(--accent-warning)` left border (4px), padding 12px 16px
- Copy: "N questions unanswered. Unanswered questions will be marked wrong." + two buttons: "Submit Anyway" (accent danger style) and "Keep Going" (ghost)
- When all answered: submit immediately without confirmation

Auto-submit on timer expiry: no confirmation — immediate transition to results, brief "Time's up" overlay for 1500ms before results render. Overlay: full-screen `var(--bg-primary)` at 0.9 opacity, centered "Time's Up" text 28px weight 700, then fade out into results view.

### View 3: Results (`type TestView = 'results'`)

Layout: Centered column, max-width 640px, gap 32px between sections.

Sections (top to bottom):
1. Title block — "Test Complete" 24px weight 700 + subject name 14px `var(--text-secondary)`
2. Score ring — reuse `.score-ring` + `.score-ring-inner` CSS classes; percentage inside, "score" label
3. Score metadata row — "N / M correct · Projected AP Score: X" — 14px `var(--text-secondary)` with projected score as a pill badge (accent or warning depending on score)
4. Per-unit breakdown card — `var(--bg-card)` border card; rows of unit name + accuracy bar + percentage
   - Unit name: 14px weight 500
   - Accuracy bar: 4px height, `var(--mastery-empty)` track, `var(--mastery-fill)` fill, 200px max-width, `transition: width 400ms ease`
   - Percentage: 13px `var(--text-muted)`, right-aligned
5. Missed questions card — same pattern as MCQResults: top 5 shown, "+ N more" if needed
6. CTA stack (column, gap 12px):
   - "Retake Test" — full-width primary accent button, Lucide `RotateCcw` icon
   - "Practice by Unit" — full-width ghost button (border `var(--bg-border)`, hover border `var(--accent)`), Lucide `BookOpen` icon
   - "Back to [Subject]" — full-width ghost button, Lucide `ArrowLeft` icon

Projected AP score pill badge colors:
- Score 5: `var(--accent-success)` tint
- Score 4: `var(--accent)` tint
- Score 3: `var(--accent-warning)` tint
- Score 1–2: `var(--accent-danger)` tint

## Interaction Contracts

### Timer
- `setInterval` at 1000ms tick; `useRef` for interval ID to prevent stale closure (established pattern)
- Timer value stored in React state as total seconds remaining
- Format: `Math.floor(seconds / 60).toString().padStart(2, '0') + ':' + (seconds % 60).toString().padStart(2, '0')`
- Warning threshold: 300 seconds (5 minutes)
- On warning: color transitions from `var(--text-primary)` to `var(--accent-danger)` in 300ms ease; optional CSS pulse animation (keyframe: opacity 1 → 0.6 → 1, 1200ms loop) — must be disabled via `prefers-reduced-motion`
- On expiry (seconds === 0): clear interval, call submit handler immediately
- Timer cleared in `useEffect` cleanup function to prevent memory leaks

### Question Navigation
- Clicking any numbered cell in nav grid jumps directly to that question (no transition — instant swap)
- Prev/Next arrows: disabled styling (opacity 0.4, `cursor: not-allowed`) when at first/last question
- Answered state is tracked as `Record<string, MCQAnswer>` — same shape as MCQSession but without auto-advance
- MCQCard receives current question; does not auto-advance on answer — student navigates manually
- MCQCard `onNext` is not used in test session; post-answer navigation is student-initiated only

### Flag for Review
- Flagged state: `Record<string, boolean>` keyed by question ID
- Flag button toggles boolean — no confirmation required
- Flagged questions shown in nav grid with warning color indicator and Flag icon overlay
- Flagged status is not written to localStorage — session-only state

### SessionStorage Guard (optional, Claude's discretion)
- If sessionStorage is available, serialize `{ answers, flagged, currentIndex, timeRemaining }` after every answer and navigation change
- On mount: hydrate from sessionStorage if key `ascendly_test_session_[subject]` exists (same-tab recovery only)
- On submit/complete: clear the sessionStorage key immediately
- Serialization must be fire-and-forget wrapped in try/catch; never block UI on storage failure

### Answer + Score Handling
- MCQCard `onAnswer` callback updates answers map; pointer-events none on choices post-submit (established pattern — Critical Rule: no per-choice disabled overhead)
- On test submit: compute per-unit accuracy from answers map + question unit metadata
- Per-unit accuracy: `correctInUnit / totalInUnit`; questions must carry their source unit slug in their data or in the loaded session metadata
- `completedRef` pattern prevents double-fire of `handleTestComplete` under React Strict Mode (established pattern from MCQResults/DrillResults)

## Animation Rules

All from MASTER.md — no new animation types introduced.

| Interaction | Duration | Easing | Property |
|-------------|----------|--------|----------|
| Button hover lift | 150ms | ease | transform translateY(-1px) |
| Button active return | 150ms | ease | transform translateY(0) |
| Card hover background | 150ms | ease | background-color |
| Progress bar fill | 400ms | ease | width |
| Timer warning color | 300ms | ease | color |
| Timer pulse (warning) | 1200ms loop | ease | opacity (1 → 0.6 → 1) |
| Results accuracy bars | 400ms | ease-out | width (entrance animation) |
| "Time's Up" overlay fade | 300ms | ease-in | opacity |
| Nav grid cell state change | 150ms | ease | background-color, border-color |

Motion: All animations except `transition: none` overrides must respect:
```css
@media (prefers-reduced-motion: reduce) {
  /* disable transitions, pulse animation, progress bar animation */
}
```

Responsive breakpoints (from MASTER.md):
- 375px (mobile): nav grid collapses to scrolling strip; timer in secondary position

- 1440px (wide): same as 1024px, `max-w-6xl` shell

Desktop session layout (>= 1024px):
- Left sidebar: TestNavGrid fixed width 240px, height 100% sticky within content area
- Main area: question card + nav row, fills remaining width
- Session header spans full width above both columns
