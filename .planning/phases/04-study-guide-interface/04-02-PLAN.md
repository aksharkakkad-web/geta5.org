    Implementation:
    1. On mount, get subject info via `getSubject(subject)` from utils/subjects.ts
    2. For each unit, check availability: `fetch(\`/data/${subject}/study-guide/unit-${unit.number}.json\`, { method: 'HEAD' })` — store result.ok in state `Record<number, boolean>`
    3. Show loading state while checks are in progress (same pattern as MCQ UnitSelector: centered "Loading study guides..." text)

    Create `app/[subject]/study-guide/page.tsx` — 'use client' page component.

    Follow the exact pattern from `app/[subject]/practice/page.tsx`:
    - Import `{ useState }` from 'react' and `{ use }` from 'react'
    - Props: `{ params: Promise<{ subject: string }> }` — extract subject via `const { subject } = use(params)`

    State machine with 2 views:
    ```typescript
    type StudyGuideView = 'unit-select' | 'reading'
    const [view, setView] = useState<StudyGuideView>('unit-select')
    const [guide, setGuide] = useState<StudyGuide | null>(null)
    const [loading, setLoading] = useState(false)
    ```

    handleSelectUnit function:
    1. Set loading to true
    2. Call `fetchStudyGuide(subject, unitNumber)` from utils/studyGuide.ts
    3. If result is null, set loading false (unit has no data — should not happen since UnitSelector checks availability, but defensive)
    4. If result exists: set guide to result, set view to 'reading', set loading false
    5. Fire analytics event (per GUIDE-04): `logEvent({ event_type: 'study_guide_view', subject, unit: \`unit-${unitNumber}\` })` — fire-and-forget, never block UI

    handleBack function:
    - Set guide to null, set view to 'unit-select'

    Render:
    ```tsx
    <main style={{ padding: '24px', maxWidth: '960px', margin: '0 auto' }}>
      {view === 'unit-select' && !loading && (
        <StudyGuideUnitSelector subject={subject} onSelectUnit={handleSelectUnit} />
      )}
      {loading && (
        <div style={{ display: 'flex', justifyContent: 'center', padding: '48px 0' }}>
          <span style={{ color: 'var(--text-muted)', fontSize: '1rem' }}>Loading study guide...</span>
        </div>
      )}
      {view === 'reading' && guide && (
        <StudyGuideReader guide={guide} subject={subject} onBack={handleBack} />
      )}
    </main>
    ```

    Note: maxWidth should be wider when in reading view to accommodate sidebar layout. Use conditional: `maxWidth: view === 'reading' ? '1100px' : '960px'`.
