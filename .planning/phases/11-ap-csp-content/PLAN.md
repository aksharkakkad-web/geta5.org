**Stimulus distribution:**
- ~47 questions with stimulus (~85%)
- ~8 without stimulus
- Stimulus types: short pseudocode segments, program descriptions, scenario text

**Multi-select allocation:** 4 multi-select questions (choose 2)

### Drill Plan
**Definition-to-term cards (22 typed-recall):**
- computing innovation, iterative development, incremental development, collaboration, pair programming, user interface, input, output, event, event-driven programming, syntax error, logic error, run-time error, debugging, documentation, comment, program, code segment, behavior (of program), development process, open source, Creative Commons

- Identify innovation type (physical/nonphysical), match error type to description, identify development process phase, input vs output identification, collaboration role scenarios, documentation purpose, testing strategies, event-driven vs sequential

**is_key_term: true candidates (10):**
computing innovation, iterative development, pair programming, user interface, event-driven programming, syntax error, logic error, debugging, program, documentation

## CB Pseudocode Examples for MCQ Stimuli

All code examples MUST use CB pseudocode only. Representative patterns:

**Variable/assignment:**
```
x ← 5
y ← x + 3
DISPLAY(y)
```

**Conditional:**
```
IF(score ≥ 90)
{
  grade ← "A"
}
ELSE
{
  grade ← "B"
}
```

**Loop:**
```
count ← 0
REPEAT UNTIL(count = 5)
{
  count ← count + 1
}
DISPLAY(count)
```

**List traversal:**
```
sum ← 0
FOR EACH num IN numList
{
  sum ← sum + num
}
DISPLAY(sum)
```

**Procedure:**
```
PROCEDURE double(n)
{
  RETURN(n * 2)
}
result ← double(7)
DISPLAY(result)
```
